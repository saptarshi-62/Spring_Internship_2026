# Data Description

This folder stores the ILPD dataset and the maintained data artifacts generated by the active notebook workflow.

## Folder Roles

### `data/raw/`

Contains the original untouched dataset.

### `data/interim/`

Contains temporary or intermediate datasets generated during preprocessing.

### `data/processed/`

Contains final processed datasets that are ready for downstream analysis, modeling, and synthetic-noise experiments.

## Current Files

### `data/raw/ILPD.csv`

The raw Indian Liver Patient Dataset used as the source for preprocessing and exploratory analysis.

### `data/interim/ILPD_cleaned.csv`

The cleaned dataset produced by `scripts/03_preprocessing_pipeline.ipynb` after schema standardization, validation, duplicate removal, and missing-value treatment.

### `data/processed/ILPD_clinically_capped.csv`

The clinically bounded version of the cleaned dataset, where implausible extremes are capped using domain-informed limits.

### `data/processed/ILPD_robust_scaled_features.csv`

The robust-scaled feature dataset built on top of the clinically capped dataframe.

### `data/processed/ILPD_robust_scaled_with_gmm_noise.csv`

The augmented robust-scaled dataset generated by `scripts/04_noise_generation_gmm.ipynb` using GMM-based synthetic rows and a `Synthetic_Flag` column.

## Workflow Link

The active notebook-to-data relationship is:

1. `scripts/03_preprocessing_pipeline.ipynb` creates `ILPD_cleaned.csv`, `ILPD_clinically_capped.csv`, and `ILPD_robust_scaled_features.csv`.
2. `scripts/02_outlier_handling.ipynb` reads `data/interim/ILPD_cleaned.csv`.
3. `scripts/04_noise_generation_gmm.ipynb` reads `data/processed/ILPD_robust_scaled_features.csv` and writes `ILPD_robust_scaled_with_gmm_noise.csv`.
4. `scripts/01_eda_univariate_bivariate.ipynb` reads `data/raw/ILPD.csv`.
